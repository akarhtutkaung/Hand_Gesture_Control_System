# Gesture Car — Hand Gesture Control System

A real-time dual-hand gesture detection system using a Mac webcam. The left hand controls movement via gesture classification; the right hand controls steering via wrist rotation. Commands are sent to a Raspberry Pi (GPIO) or Webots simulator (UDP). An auth flow (facial recognition + unlock gesture) guards entry before control starts.

---

## Supported Gestures

| Hand  | Gesture    | How to make it                                              | Command          |
|-------|------------|-------------------------------------------------------------|------------------|
| Left  | Palm       | Open hand flat, all fingers spread out                      | FORWARD          |
| Left  | Fist       | Close all fingers tightly                                   | STOP             |
| Left  | Peace      | Index and middle finger up, others folded                   | BACKWARD         |
| Right | Hold Fist  | Close fist to activate steering, then rotate wrist left/right | STEER:\<angle\> |

---

## What It Uses

- **MediaPipe HandLandmarker** — 21-point hand landmark detection via the Tasks API
- **OpenCV** — webcam capture and frame display
- **scikit-learn** — SVM classifier trained on recorded gesture samples
- **NumPy** — landmark array manipulation

## Setup

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
mkdir -p data model
```

> Apple Silicon: use `pip install mediapipe-silicon` instead of `mediapipe`.

Download the MediaPipe hand landmark model (required — one time only):

```bash
curl -L -o model/hand_landmarker.task \
  https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/1/hand_landmarker.task
```

macOS will prompt for camera permission on first run — click Allow.

## How to Run

Run all commands from the project root.

### Step 1 — Collect gesture samples

```bash
python src/controller/collect_data.py
```

| Key | Gesture | Command  |
|-----|---------|----------|
| `f` | Fist    | STOP     |
| `p` | Palm    | FORWARD  |
| `v` | Peace   | BACKWARD |
| `q` | —       | Quit     |

Aim for ~100 samples per gesture. Data saves automatically to `data/`.

### Step 2 — Train the classifier

```bash
python src/controller/train_model.py
```

Reads `data/fist.csv`, `data/palm.csv`, and `data/peace.csv`, trains an SVM pipeline, and saves the model to `model/gesture_classifier.pkl`. Prints cross-validation accuracy.

### Step 3 — Run the app

```bash
python src/app.py
```

Runs the auth flow (facial recognition → unlock gesture), then opens the webcam and starts dual-hand gesture control. Press `q` to quit.

## Deployment

### Raspberry Pi

Set the `USE_SOCKET=1` environment variable to route commands over UDP, or implement the GPIO stubs in `src/comms/car_interface.py` (`ON_PI` is detected automatically).

```bash
pip install -r requirements.txt
pip install RPi.GPIO   # or gpiozero
python src/app.py
```

### Webots Simulator

Start the controller from `src/others/Webots/`, then run the app with UDP enabled:

```bash
USE_SOCKET=1 python src/app.py
```

The Webots controller listens on `127.0.0.1:5005` (configurable via `SOCKET_HOST` / `SOCKET_PORT` in `config.py`).
